# 产品文档（当前功能点总览）

本文用于记录项目“当前已经实现/可体验”的功能点清单。后续任何功能点新增、删除、修改，都需要同步更新本文。

## 入口与加载

- 入口：`index.html` 启动 Egret，入口类 `Main`（见 `src/main.ts`）。
- 资源加载：`RES.loadConfig(resource/default.res.json)` + `RES.loadGroup("preload")`。
- FairyGUI 包加载：加载 `base_fui`、`dule_fui` 以及 `dule` 的 3 个图集资源（见 `resource/default.res.json` 与 `src/main.ts`）。
- 卡牌配置加载：把 `resource/config/cards/*.json` 注入到 `ConfigManager` 的 `cards` 配置（见 `src/main.ts`）。
- 分辨率口径：以 `2048x1366` 为设计分辨率，不做缩放适配（见 `src/main.ts` 与 `src/project/dule/DuleView.ts`）。

## Dule 对战 Demo（最低可运行）

### 战斗装配（伪代码结构）

- `DuleView.open()` 负责装配战斗：创建 `DuleBattle`、注册 Actor、注册 Rule、铺桌面 Zone 数据、调用 `changeState("game_start")` 启动（见 `src/project/dule/DuleView.ts`）。
- Actor：`PlayerActor` / `AiPlayerActor` / `GameMasterActor` 为占位实现（见 `src/project/dule/actors/*`）。
- Rule：`DuleBasicRule` 监听 `game_start`，触发抽牌/回合开始的 Action 流程（见 `src/project/dule/rules/DuleBasicRule.ts`）。
- 卡牌工厂：`CardFactory.createCardById()` 支持 `hero_001`、`001/002/003`（见 `src/project/dule/CardFactory.ts`）。

### 数据结构（Zone）

- 核心 Zone：`hero_*`、`hand_*`、`deck_*`、`field_*`、`mana_*`（见 `src/project/dule/DuleBattle.ts`）。
- 武器 Zone：`weapon_me`、`weapon_other`，并提供别名 `weapon_zone` 指向 `weapon_me`（见 `src/project/dule/DuleBattle.ts`）。
- Zone 顶部卡：`Zone.getTopCard()`（见 `src/cardgame/Zone.ts`）。
- 费用（水晶）：开局 `0/0`，每回合开始最大值 +1 并回满，上限 10（见 `src/project/dule/actions/TurnStartAction.ts`）。

## 交互（手牌）

- 点击选中/取消选中：
  - 点击一张手牌：摆正 + 放大 + 黄色外发光
  - 再次点击同一张：取消选中并回到弧形布局
  - 点击手牌以外区域：取消选中（点在选中卡牌自身不会触发取消，避免闪烁）
- 拖拽出牌（不改变手牌排序）：
  - 开始拖拽即取消选中
  - 拖拽进入“触发状态”时显示蓝色外发光
  - 松开：触发状态则尝试打出；否则回到原位
  - 出牌动画：打出瞬间放大后快速缩小消失（对方出牌也播放）
- 可释放提示：
  - 我方回合且费用足够时，手牌卡牌常驻绿色外发光提示可释放
  - 选中（黄色）与拖拽触发态（蓝色）优先级更高，会覆盖该提示
- 法术瞄准箭头（单位目标法术）：
  - 当拖拽进入“触发状态”，且该法术 `targetType=unit` 且当前回合费用足够时：手牌卡牌完全透明
  - `ArrowTargetView` 显示瞄准箭头并跟随鼠标移动
  - 鼠标悬停在可施放单位（当前实现：对方英雄或对方场上随从）时，显示 target 标记
  - 触发状态松开：若命中有效单位则按该目标出牌，否则视为未出牌并回到原位
- 随从攻击瞄准（可行动随从）：
  - 我方随从 `exhausted=0` 且当前回合时，可拖拽小范围进入“瞄准态”
  - 进入瞄准态后显示 `ArrowTargetView`，并跟随鼠标；悬停对方英雄/对方随从时显示 target 标记
  - 松开：若命中有效目标则触发随从攻击（`AttackAction`），否则不触发

实现位置：`src/project/dule/views/HandView.ts`。
箭头实现：`src/project/dule/views/ArrowTargetView.ts`，事件链路为 `HandView/MinonView -> 事件 -> DuleView -> ArrowTargetView(show/move)`。

## 交互（英雄技能）

- 英雄技能卡：`hero_fireball.json`，作为一张技能卡绑定到英雄技能按钮。
- 使用条件：我方回合、费用足够、使用次数 > 0。
- 使用次数：默认 1 次，次数不足时按钮置灰。
- 交互：
  - 长按：显示技能说明 tip。
  - `targetType` 为空：点击立即施放。
  - `targetType=unit`：拖动进入瞄准态，松开命中目标后施放。

## 动画序列（Action Animation）

- Flow 执行后派发 `after-*` 事件，UI 监听并投递到 `ActionAnimationManager` 队列（见 `src/project/dule/animations/ActionAnimationManager.ts`）。
- 动画分为阻塞/非阻塞，以及是否等待下一条执行；当前已接入抽牌、出牌与随从入场动画（缩放缓动）。
- 手牌显示采用“延迟提交”以避免一次性刷出所有新牌。

## 卡牌脚本（JASS）

- 卡牌配置：`resource/config/cards/*.json` 支持 `tags`，其中 `tag/trigger/script` 用于声明脚本能力（示例：`fireball.json`）；配置的 `tags` 统一转为 `ScriptTag` 执行。
- 触发点：
  - `onPlay`：出牌时执行（`PlayCardAction` 调用 `card.triggerTagsByType("onPlay", ctx)`）。
  - `onEnable`：随从入场/卡牌进入生效状态时执行（`DuleBaseCard.enable()` 触发 Tag 生命周期；见 `src/project/dule/tags/ScriptTag.ts`）。
- 执行引擎：`DuleBattle` 初始化 `DuleJassEngine`，底层 JASS 引擎来自 `tiny.common/scripts`（见 `src/project/dule/script/DuleJassEngine.ts`）。
- 内置脚本函数（最低实现）：
  - `select_zone_card_by_tag('unit')`：读取当前出牌目标并写入脚本上下文 selected
  - `task_spell_damage(n)`：对 selected 或默认 target 造成伤害
  - `on(eventName, script)`：运行时订阅任意事件，并在 Tag `onDisable/onDispose` 时自动退订（见 `src/project/dule/tags/ScriptTag.ts`）
- 随从入场：若脚本 tag 附着于随从卡牌，入场时会复用该卡牌实例以保留 tag（见 `src/project/dule/actions/SummonMinionAction.ts`）。
- 出牌顺序：随从类 `onPlay` 召唤流程在“移除手牌/扣费”之后入队，保证出牌动画先于入场动画。

## 已知限制（当前实现口径）

- 暂无。
- 出牌目标：
  - 非单位目标法术/随从：当前仍默认以对方英雄为目标（`targetZone=hero,targetIndex=0`）
